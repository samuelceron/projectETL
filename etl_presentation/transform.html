Transformation process

In the data transformation stage, a series of rules or functions are applied to the extracted data in order to prepare it for loading into the end target: THE DATABASE

The key point in this process is the DATABASE NORMALIZATION in order to reduce data redundancy and improve data integrity, among other benefits we'll see in our conclusions.

At this point the team look for the Third Normal Form (3NF).

For that proposal, we made some basic transformations in python (Jupyter-Notebook):

  • First, we eliminate unnecesary columns from the main DataFrame (Games)
  • Then, for the owners column we create a unique catalogue and replace values with ids


  • Second, we cleaned the developer column, as the main developer appears first, we keep the first value and then replace it with id numbers
  • We created a dictionary and changed text values for id of developers
  • We applied the same criteria to the columns Publiser, categories and steampspy_tags




  • Third, for platform we saw there are no many categories for this column






Finally, we see the resulting table that will be loaded along with the other ones we created

The result is a junction table
